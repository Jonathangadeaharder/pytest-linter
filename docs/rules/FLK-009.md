# PYTEST-FLK-009: SubprocessWithoutTimeoutRule

**Category:** Flakiness
**Severity:** Warning

## Description

Detects usage of `subprocess.Popen`, `subprocess.run`, and similar calls without a
`timeout` parameter. Without a timeout, tests may hang indefinitely.

## Why This Matters

- Subprocess calls can hang if the child process deadlocks
- CI runners have limited time budgets
- Tests should fail fast, not hang

## Example

```python
# Bad
def test_command():
    result = subprocess.run(["my-command"])
    assert result.returncode == 0

# Good
def test_command():
    result = subprocess.run(["my-command"], timeout=30)
    assert result.returncode == 0
```

## How to Fix

1. Add `timeout=30` (or appropriate value) to subprocess calls
2. Use `subprocess.run(..., timeout=30, check=True)`
3. Wrap in `pytest.mark.timeout()` for the test function
