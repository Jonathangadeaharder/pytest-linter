# PYTEST-FLK-001: TimeSleepRule

**Category:** Flakiness
**Severity:** Warning

## Description

Detects the use of `time.sleep()` in test functions. Sleep calls make tests slow and flaky
because they depend on timing assumptions that may not hold in CI environments.

## Why This Matters

- Tests with `time.sleep()` are inherently slow
- Timing assumptions break under load or on different hardware
- CI environments have variable performance

## Example

```python
# Bad
def test_api():
    time.sleep(2)  # Waits for server
    result = make_request()
    assert result.ok

# Good
def test_api():
    result = wait_for_condition(make_request, timeout=5)
    assert result.ok
```

## How to Fix

1. Use `pytest.mark.timeout()` for timeout-based tests
2. Use polling/wait-for-condition patterns
3. Mock time-dependent behavior
4. Use `asyncio.sleep()` with proper async test patterns
