# PYTEST-FLK-008: RandomWithoutSeedRule

**Category:** Flakiness
**Severity:** Warning

## Description

Detects usage of the `random` module without calling `random.seed()`. Tests that use
random values without a fixed seed produce non-reproducible results.

## Why This Matters

- Tests may pass locally but fail in CI
- Debugging is difficult without reproducible random sequences
- Random test data should be deterministic for reliability

## Example

```python
# Bad
def test_random():
    value = random.randint(1, 100)
    assert value > 0

# Good
def test_random():
    random.seed(42)
    value = random.randint(1, 100)
    assert value > 0
```

## How to Fix

1. Call `random.seed()` at the start of the test
2. Use a fixture that sets a fixed seed
3. Use `hypothesis` for property-based testing with controlled randomness
